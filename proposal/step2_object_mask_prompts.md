# Step 2: Object Mask Prompts

**Duration**: ~1 week
**Depends on**: Step 1 (interaction decoder with contact tokens)
**Goal**: Condition contact prediction on an object mask, enabling semantic/object-aware contact

---

## Motivation

DECO and BSTRO predict contact without knowing *what* the person is touching. But contact patterns are object-dependent: holding a cup involves hand contact, sitting on a chair involves buttocks/back/thighs. InteractVLM showed that conditioning contact on object identity significantly improves predictions ("Semantic Human Contact").

By adding an object mask prompt, the model can learn object-specific contact priors: "when there's a chair in this region, expect sitting contact."

## New Components

### Object Mask Prompt Encoder
- Input: Binary object mask M_obj (HxW) from SAM or manual annotation
- Encoding: Same mask encoder as SAM3DB uses for human masks, but with a different **type embedding** to distinguish "human mask" from "object mask"
- Output: Mask tokens projected to d_model

### Object Query Tokens
- 1-2 learnable object query tokens added to interaction decoder
- These attend to image features F and to the object mask prompt tokens
- They aggregate object identity and location information
- Contact tokens cross-attend to object tokens (via self-attention in the decoder)

### Semantic Contact Head (Optional)
- From contact tokens, predict not just where contact is, but the **type**:
  - Contact type: {support, grasp, lean, press, step, sit, rest}
  - Target type: {object, floor, wall, furniture}
- Training: DAMON provides object class labels (84 categories) per contact annotation

---

## Training

**What trains**: Object mask encoder + object query tokens + interaction decoder (finetuned) + semantic head
**What's frozen**: SAM3DB encoder + body decoder

**Data**:
| Dataset | Images | Object Masks | Usage |
|---------|--------|-------------|-------|
| DAMON | ~4.4K | Generated by running SAM on images | Primary -- masks + contact labels |
| RICH | ~577K | Dense body-scene contact (proximity-based) | Additional 3D contact supervision, regularizes overfitting |

**Object mask generation for DAMON**: Run SAM on DAMON images to segment objects. DAMON already has object class labels (84 COCO categories), so we can match SAM masks to labeled objects.

**Training modes** (randomly sampled per batch):
1. **With object mask** (70%): Provide object mask prompt. Contact should improve.
2. **Without object mask** (30%): Drop the object mask. Contact falls back to generic mode (like DECO). This ensures the model works without prompts.

**Loss**:
```
L = L_bce + lambda_sem * L_semantic

L_semantic: Cross-entropy on contact type + target type predictions
            Only active when object mask prompt is provided
```

**Key evaluation**: InteractVLM's "Semantic Human Contact" benchmark on DAMON. They report per-category F1 across 7 object super-categories (Accessory, Daily Obj, Food, Furniture, Kitchen, Sports, Transport).

---

## Ablations

1. Object mask prompt vs. no prompt (measure contact improvement)
2. Object category embedding vs. mask-only (does knowing "chair" vs. just seeing the mask help?)
3. 1 vs. 2 object query tokens
4. Prompt dropout rate: 0% vs. 30% vs. 50%

---

## Success Criteria

- F1 with object mask > F1 without object mask (by at least 2-3 points)
- F1 without object mask >= Step 1 baseline (no degradation)
- Qualitative: with chair mask, model predicts sitting contact; with cup mask, model predicts hand contact
- Semantic contact accuracy competitive with or exceeding InteractVLM on DAMON
